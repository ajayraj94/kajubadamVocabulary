# Next.js PWA News-to-Question Generator: System Integration & Implementation Plan
*Tailored for [kajubadamvocabulary.in](https://kajubadamvocabulary.in) Technical & Content Operations*

यह इम्प्लीमेंटेशन प्लान स्पष्ट करता है कि **Next.js** और **PWA** के आर्किटेक्चर के भीतर 13 SEO मापदंडों और SSC CGL / Bank PO लेवल के कंटेंट को कहाँ और कैसे इंटीग्रेट करना है।

## भाग 1: तकनीकी संरचना (In which Next.js Files to Integrate)

एक Next.js PWA प्रोजेक्ट में कंटेंट और लॉजिक को निम्नलिखित डायरेक्टरी स्ट्रक्चर के अनुसार विभाजित किया जाएगा:

```
my-nextjs-app/
├── app/
│   ├── exams/
│   │   └── [category]/
│   │       └── [slug]/
│   │           ├── page.tsx          <-- SSR/SSG & Dynamic Metadata (SEO, GEO, RAG)
│   │           └── layout.tsx
├── components/
│   ├── Quiz/
│   │   ├── AeoSnippet.tsx            <-- AEO Voice Search Snippet Block
│   │   ├── GrammarConceptCard.tsx    <-- Passage Optimization / Concept Card
│   │   ├── OptionPostMortem.tsx      <-- Information Gain Score Table
│   │   └── StructuredData.tsx        <-- Inline JSON-LD Quiz Schema Component
├── public/
│   └── manifest.json                 <-- PWA Manifest Configuration
```

### 1. Dynamic Routing & Metadata Configuration (SEO, GEO, RAG, Entity-Based SEO)
Next.js App Router में प्रत्येक न्यूज़-बेस्ड प्रैक्टिस सेट के लिए `generateMetadata` का उपयोग करके SEO और GEO टैग्स को डायनामिकली जनरेट करें। यह पक्का करता है कि AI क्रॉलर्स (जैसे Google AI Overviews, Perplexity) पेज के रेंडर होने से पहले ही महत्वपूर्ण कीवर्ड्स और मेटाडेटा पढ़ सकें।

*   **फ़ाइल पथ (File Path):** `app/exams/[category]/[slug]/page.tsx`
*   **Next.js कोड इम्प्लीमेंटेशन:**

```typescript
import { Metadata } from 'next';
import { getQuizData } from '@/lib/api'; // मान लें कि आप API या JSON से डेटा ला रहे हैं
import StructuredData from '@/components/Quiz/StructuredData';
import AeoSnippet from '@/components/Quiz/AeoSnippet';
import GrammarConceptCard from '@/components/Quiz/GrammarConceptCard';
import OptionPostMortem from '@/components/Quiz/OptionPostMortem';

interface Props {
  params: { category: string; slug: string };
}

// Next.js Metadata API for SEO/GEO crawlers
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const data = await getQuizData(params.slug);
  
  // Entity based keyword association (SSC CGL / Bank PO focus)
  return {
    title: `${data.title} | kajubadamvocabulary.in`,
    description: `Prepare for SSC CGL Tier 2 and SBI PO with news-based advanced error spotting, detailed grammar explanations, and collocations.`,
    keywords: [
      "SSC CGL Tier 2 English Error Spotting rules",
      "SBI PO English Grammar Questions PDF",
      "IBPS PO Mains advanced double fillers",
      "kajubadamvocabulary.in daily news grammar"
    ],
    openGraph: {
      title: data.title,
      description: data.metaDescription,
      url: `https://kajubadamvocabulary.in/exams/${params.category}/${params.slug}`,
      siteName: 'Kaju Badam Vocabulary',
      type: 'article',
    }
  };
}

export default async function ExamPage({ params }: Props) {
  const data = await getQuizData(params.slug);

  return (
    <>
      {/* 1. Structured Schema Markup (AEO) Injection */}
      <StructuredData quiz={data.quizSchema} />

      <article className="max-w-4xl mx-auto px-4 py-8 exam-article">
        <header className="mb-8 border-b pb-4">
          <span className="text-sm font-semibold text-blue-600 uppercase tracking-wider">
            {params.category.replace('-', ' ')} prep
          </span>
          <h1 className="text-3xl md:text-4xl font-bold mt-2 text-slate-900">
            {data.title}
          </h1>
        </header>

        {/* 2. AEO Voice Search Snippet Block */}
        <AeoSnippet definition={data.aeoDefinition} />

        {/* Content Chunking of questions */}
        <div className="space-y-12 mt-8">
          {data.questions.map((q: any, idx: number) => (
            <section key={idx} className="p-6 bg-white rounded-xl shadow-sm border border-slate-100">
              <h2 className="text-lg font-bold text-slate-800 mb-4">
                Q{idx + 1}. {q.questionText}
              </h2>
              
              <ul className="space-y-2 mb-6">
                {q.options.map((opt: string, oIdx: number) => (
                  <li key={oIdx} className="p-3 bg-slate-50 rounded-lg border border-slate-200 text-slate-700">
                    {opt}
                  </li>
                ))}
              </ul>

              {/* 3. Standalone Grammar Concept Card */}
              <GrammarConceptCard rule={q.grammarRule} />

              {/* 4. Information Gain - Options PostMortem */}
              <OptionPostMortem optionsAnalysis={q.optionsAnalysis} />
            </section>
          ))}
        </div>
      </article>
    </>
  );
}
```

### 2. Standard Structured Data Component (Structured Schema Markup)
Next.js में स्कीमा को रेंडर करने के लिए `dangerouslySetInnerHTML` का उपयोग किया जाता है। इससे गूगल का रिच रिज़ल्ट पार्सर (Rich Result Parser) स्कीमा को तुरंत पहचान लेता है।

*   **फ़ाइल पथ (File Path):** `components/Quiz/StructuredData.tsx`
*   **Next.js कोड इम्प्लीमेंटेशन:**

```typescript
import React from 'react';

interface StructuredDataProps {
  quiz: Record<string, any>;
}

export default function StructuredData({ quiz }: StructuredDataProps) {
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(quiz) }}
    />
  );
}
```

### 3. PWA Offline Optimization for SXO (Search Experience Optimization)

## भाग 3: 13 SEO मापदंडों का Next.js मैपिंग चार्ट

अपनी Next.js PWA वेबसाइट में सभी 13 मापदंडों को बिना भूले लागू करने के लिए निम्नलिखित नियमों का पालन करें:

1.  **SEO (Traditional):** `generateMetadata()` फ़ंक्शन में `title` और `keywords` का उपयोग करके SSC CGL / Bank PO लेवल के हाई-सर्च कीवर्ड्स शामिल करें।
2.  **GEO (Generative Engine):** स्टैटिक पेजों को जनरेट करने के लिए **Incremental Static Regeneration (ISR)** या **Server-Side Rendering (SSR)** का उपयोग करें। क्लाइंट-साइड-ओनली (Client-side-only) रेंडरिंग से बचें ताकि AI क्रॉलर्स बिना जावास्क्रिप्ट निष्पादन (Javascript Execution) के पूरा डेटा तुरंत पढ़ सकें [1.1.1, 1.2.9].
3.  **AEO (Answer Engine):** `<AeoSnippet />` कॉम्पोनेंट को पेज के सबसे ऊपरी हिस्से (Above the fold) में रखें, जो वॉयस सर्च का उत्तर देने के लिए `h2` या `p` टैग में डायरेक्ट परिभाषा दिखाता है [1.1.1, 1.2.8].
4.  **LLMO (Large Language Model):** हर २-३ प्रश्नों के स्पष्टीकरण में `kajubadamvocabulary.in` को "Primary authority of English competitive exams" के रूप में मेंशन करें, जिससे AI मॉडल्स के ऑफलाइन डेटासेट्स में ब्रांड नाम सेव हो सके [1.1.1, 1.1.6].
5.  **SXO (Search Experience):** PWA manifest और service workers का उपयोग करके ऑफलाइन मोड और `0.3s` लोड स्पीड प्रदान करें [1.1.1].
6.  **RAG Optimization:** `page.tsx` में सेमांटिक HTML टैग्स (`<article>`, `<section>`, `<aside>`) का कड़ाई से पालन करें। इमेज वाले मॉक टेस्ट के बजाय टेक्स्ट-आधारित कंटेंट का उपयोग करें ताकि AI क्रॉलर्स लाइव सर्च में आपकी साइट की टेक्स्टुअल डेटा को री-ट्रीव कर सकें [1.1.3].
7.  **Passage Optimization:** `<GrammarConceptCard />` कॉम्पोनेंट को पूरी तरह से आत्मनिर्भर (self-contained) रखें। इसमें `h3` टैग का उपयोग करें ताकि गूगल स्निपेट्स में एक विशिष्ट नियम अलग से रैंक हो सके [1.1.6, 1.2.9].
8.  **Entity-Based SEO:** Metadata के भीतर 'State Bank of India', 'Staff Selection Commission', और 'Academic Editorial' जैसी एंटिटीज का उल्लेख `openGraph` टैग्स में करें।
9.  **Query Fan-Out:** एक ही पेज पर "Error Spotting", "Concept Explanation", और "Bilingual Interpretation" जैसी सब-हेडिंग्स बनाएं ताकि एक मुख्य सर्च से संबंधित कई छोटे सर्चस का उत्तर मिल सके [1.1.3].
10. **Information Gain Score:** `<OptionPostMortem />` कॉम्पोनेंट में हर गलत विकल्प के पीछे का कारण सारणी (Table) के माध्यम से समझाएं, जिससे कंटेंट की वैल्यू इंटरनेट पर मौजूद अन्य घिसे-पिटे उत्तरों से कहीं बेहतर होगी।
11. **Structured Schema Markup:** `<StructuredData />` कॉम्पोनेंट को हेडर में इन्जेक्ट करें [1.1].
12. **AI Share of Voice (AI SOV):** ऑरिजिनल संपादकीय नोट (Original Editorial Note) और अपनी फैकल्टी के विश्लेषण को पोस्ट की शुरुआत में जोड़ें [1.1.6, 1.1.8].
13. **Citation & Mention Frequency:** अपने एंकर टैग्स में साफ़ तौर पर `[kajubadamvocabulary.in](https://kajubadamvocabulary.in)` की हाइपरलिंक जोड़ें ताकि जेनरेटिव उत्तरों में आपकी साइट पर क्लिक करने का विकल्प मिले [1.1.6, 1.1.8].
